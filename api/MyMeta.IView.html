Interface IView

IView represents a view in your DBMS.

Namespace: MyMeta
Assembly: OpenMyMeta.dll
Syntax
[Guid("3C797523-4407-4AFC-9369-2F6F8BDF5A96")]
[InterfaceType(ComInterfaceType.InterfaceIsDual)]
public interface IView
Remarks

IView Collections:

ColumnsContains all of the columns for the view
SubTablesAn ITables collection that contains all the tables used directly by this view
SubViewsAn IViews collection that contains all of the views used directly by this view
PropertiesA collection that can hold key/value pairs of your choosing
GlobalPropertiesA collection that can hold key/value pairs of your choosing for all Views with the same Database
AllPropertiesA collection that combines the Properties and GlobalProperties Collections
Examples

VBScript

Dim objView
For Each objView in objDatabase.Views
    output.writeLn objView.Name
    output.writeLn objView.Alias
Next

JScript

var objView;
for (var j = 0; j < objDatabase.Views.Count; j++) 
{
    objView = objDatabase.Views.Item(j);

    output.writeln(objView.Name);
    output.writeln(objView.Alias);
}

Alias

You can override the physical name of the View. If you do not provide an Alias the value of 'View.Name' is returned. If your view in your DBMS is 'Q99AAB' you might want to give it an Alias of 'Employees' so that your business object names will make sense. You can provide an Alias the User Meta Data window. You can also set this during a script and then call MyMeta.SaveUserMetaData(). See Name

AllProperties

AllProperties is essentially a read-only collection consisting of a combination of both the Properties (local) collection and the GlobalProperties (global) collection. The local properties are added first, and then the global properties are added however, only global properties for which no local property exists -- are added. This makes local properties overlay global properties. Global properties can act as a default value which can be overridden by a local property. See IProperty.

GlobalProperties

The Properties for all Views within the same Database. These are user defined and are typically stored in 'UserMetaData.xml' unless changed in the Default Settings dialog. Properties consist of key/value pairs. You can populate this collection during your script or via the Dockable window. To save any data added to this collection call MyMeta.SaveUserMetaData().
