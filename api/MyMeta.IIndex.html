Interface IIndex

IIndex represents an index on a table in your DBMS.

Namespace: MyMeta
Assembly: OpenMyMeta.dll
Syntax
[Guid("A9888A00-338A-49FE-A5A7-77AD895D520E")]
[InterfaceType(ComInterfaceType.InterfaceIsDual)]
public interface IIndex
Remarks

IIndex Collections:

ColumnsA collection of columns that make up this Index
PropertiesA collection that can hold key/value pairs of your choosing
GlobalPropertiesA collection that can hold key/value pairs of your choosing for all Indexes with the same Database
AllPropertiesA collection that combines the Properties and GlobalProperties Collections
Examples

VBScript

Dim objIndex
For Each objIndex in objTable.Indexes
    output.writeLn objIndex.Name
    output.writeLn objIndex.Alias
Next

JScript

var objIndex;
for (var j = 0; j < objTable.Indexes; j++) 
{
    objIndex = objTable.Indexes.Item(j);

    output.writeln(objIndex.Name);
    output.writeln(objIndex.Alias);
}

Alias

You can override the physical name of the index. If you do not provide an Alias the value of 'Index.Name' is returned. If your column in your DBMS is 'TXT_FIRST_NAME' you might want to give it an Alias of 'FirstName' so that your business object property will be a nice name. You can provide an Alias the User Meta Data window. You can also set this during a script and then call MyMeta.SaveUserMetaData(). See Name

AllProperties

AllProperties is essentially a read-only collection consisting of a combination of both the Properties (local) collection and the GlobalProperties (global) collection. The local properties are added first, and then the global properties are added however, only global properties for which no local property exists -- are added. This makes local properties overlay global properties. Global properties can act as a default value which can be overridden by a local property. See IProperty.

FillFactor

For a B+-tree index, this property represents the storage utilization factor of page nodes during the creation of the index. The value is an integer from 0 to 100 representing the percentage of use of an index node. For a linear hash index, this property represents the storage utilization of the entire hash structure (the ratio of used area to total allocated area) before a file structure expansion occurs.
