Interface IForeignKey

IForeignKey represents an foreign key on a table in your DBMS.

Namespace: MyMeta
Assembly: OpenMyMeta.dll
Syntax
[Guid("B8E74CDA-CD66-4E90-844E-C0E120FEB0B5")]
[InterfaceType(ComInterfaceType.InterfaceIsDual)]
public interface IForeignKey
Remarks

IForeignKey Collections:

ForeignColumnsA collection of columns that are in the foreign table that make up the key
PrimaryColumnsA collection of columns that are in the parent table that make up the key
PropertiesA collection that can hold key/value pairs of your choosing
GlobalPropertiesA collection that can hold key/value pairs of your choosing for all ForeignKeys with the same Database
AllPropertiesA collection that combines the Properties and GlobalProperties Collections
Examples

VBScript

Dim objForeignKey
For Each objForeignKey in objTable.ForeignKeys
    output.writeLn objForeignKey.Name
    output.writeLn objForeignKey.Alias
Next

JScript

var objForeignKey;
for (var j = 0; j < objTable.ForeignKeys; j++) 
{
    objColumn = objTable.ForeignKeys.Item(j);

    output.writeln(objForeignKey.Name);
    output.writeln(objForeignKey.Alias);
}

Alias

You can override the physical name of the foreign key. If you do not provide an Alias the value of 'ForeignKey.Name' is returned. If your foreign key in your DBMS is 'TXT_FIRST_NAME' you might want to give it an Alias of 'FirstName' so that your business object property will be a nice name. You can provide an Alias the User Meta Data window. You can also set this during a script and then call MyMeta.SaveUserMetaData(). See Name

AllProperties

AllProperties is essentially a read-only collection consisting of a combination of both the Properties (local) collection and the GlobalProperties (global) collection. The local properties are added first, and then the global properties are added however, only global properties for which no local property exists -- are added. This makes local properties overlay global properties. Global properties can act as a default value which can be overridden by a local property. See IProperty.

GlobalProperties

The Properties for all ForeignKeys within the same Database. These are user defined and are typically stored in 'UserMetaData.xml' unless changed in the Default Settings dialog. Properties consist of key/value pairs. You can populate this collection during your script or via the Dockable window. To save any data added to this collection call MyMeta.SaveUserMetaData().
